Bug report guidelines



1) This bug tracker is for end-user issues with Blender binaries


  • Don't report problems with compiling or building Blender. Check our Get Involved page.

    2) Is it a Python Add-on?


  • Go to the Python Extension tracker to report it.

    • 3) Is it a real bug?


  • Always test with official releases from blender.org. Or get our test builds from builder.blender.org.
  • It has to be intended to work by development. (e.g. not a feature request)
  • It should be repeatable consistently.
  • Check on another computer if possible.
  • Sometimes driver or s/w upgrades cause problems, try a previous release that worked to verify this.
  • Keep updated with our official todo list.

    4) Reporting a bug


  • Always mention your OS version, 3D card type and Blender version(s) you tested.
  • Own compiled versions: include svn version number from splash screen. Don't forget to test our official release too.
  • Essential: ensure others can redo errors by giving exact instructions.
  • Don't provide long lists of actions, find the minimum amount of instructions showing how to redo it.


    Important: attach a (as simple as possible) .blend file that shows the issue!
